## Follow-up: enrichment estimator — the deck's "base × joint fraction ± CI" made real (F025)
Context: An Vy Le's "How it works" deck (TrialBridge-Workflow-Explained.pdf) describes the step-4 estimator as prose; it wasn't in the code (the site-matcher, feasibility funnel, and modeledPrevalence multiply-the-marginals path were — grep confirmed no standardize/joint/CI/transportability anywhere). A methodology review of the deck raised three specific weaknesses; this closes all three in `src/lib/enrichmentEstimator.ts` (pure, deterministic, no model in the loop — same discipline as feasibility.ts/modeledPrevalence.ts).
- [crit-1] **Joint fraction gets its OWN interval.** `wilsonInterval(k,n)` — Wilson score (not Wald) because the joint count is small and near 0, where Wald goes negative and understates width. Demonstrated: marginal HER2+ 0.18 [0.146,0.220] rel-width 41% vs JOINT 0.045 [0.029,0.070] rel-width 92% — ~2x wider, and the deck showed the joint bare. This is the number that actually drives the estimate.
- [crit-2] **Multi-covariate standardization + explicit sparse-cell policy.** `standardizeRate(strata,{minStratumN})` does direct standardization over any number of cross-classified strata (the deck's 0.6·0.20+0.4·0.15=0.18 is just the 2-stratum case), propagates Σŵ²·p(1-p)/m variance, and — the answer to "what happens as strata get sparse" — redirects a thin/empty cell's base weight onto the pooled rate (over adequate cells only), flags it, and reports `coveredWeight` so a reviewer sees how much rests on the fallback. Empty cell (n=0) handled without NaN; weight never silently dropped.
- [crit-3] **Transportability is a NUMBER, not a shrug.** `transportabilityBound(interval,tau,max)` widens the sampling CI by a stated bias factor τ, labelled an assumption (not data). São Paulo base 3200: sampling [92,224], +τ=0.3 → [64,291]. `estimateEligible` passes `base` as the ceiling (eligible ≤ base) — caught a real bug where a count band was clamped to [0,1].
- [estimator] `estimateEligible({base,jointK,jointN,tau?,strata?})` = base × fraction ± CI per region; base treated as exact so all count-band uncertainty comes from the fraction; reports raw Wilson AND standardized fraction when strata supplied.
- [tests] `tests/enrichment.test.ts` — 21 cases (Wilson correctness/boundary, joint-vs-marginal width, standardization reweighting, thin+empty cell policy, fallback-widens-interval, transportability containment, base-exact, determinism). `npm test` 88/88 (up from 67), typecheck clean, `npm run build` OK (8 routes — pure lib, nothing wired to the UI yet; that's the obvious next step: surface the joint CI + τ band on /sponsor and /scorecard).
